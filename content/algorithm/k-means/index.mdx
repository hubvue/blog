## 什么是聚类？
我们在业务中所遇到的关于类别的问题大多数都是分类问题，所谓分类问题是：确定已知的类别，将数据源根据不同类的特征分布到特定类别下。
聚类问题在机器学习中经常会出现，在业务中反而不太容易会遇到，聚类问题是事先不知道数据源会分为几类，通过聚类分析计算将数据源聚合成几个集群。

可以看出区别问题是聚类问题还是分类问题的一个明显特征是：是否确定已知类别。

## K-Means 聚类
`K-Means`是一种比较常见的、基于迭代求解的聚类分析算法。它的有点事简单、容易理解、速度快；缺点也比较明显，必须指定将数据源分为几类。

`K-Means`聚类算法经常出现在距离求解上，在知乎上看到一篇使用`牧师-村民模型`来解释`K-Means`我认为比较好理解:

> 有四个牧师去郊区布道，一开始牧师们随意选了几个布道点，并且把这几个布道点的情况公告给了郊区所有的村民，于是每个村民到离自己家最近的布道点去听课。
听课之后，大家觉得距离太远了，于是每个牧师统计了一下自己的课上所有的村民的地址，搬到了所有地址的中心地带，并且在海报上更新了自己的布道点的位置。
牧师每一次移动不可能离所有人都更近，有的人发现A牧师移动以后自己还不如去B牧师处听课更近，于是每个村民又去了离自己最近的布道点……
就这样，牧师每个礼拜更新自己的位置，村民根据自己的情况选择布道点，最终稳定了下来。

### 算法步骤
`K-Means`聚类算法的作用是数据源分为k类，算法步骤如下：
1. 确定k的值，表示将数据源分为k组；
2. 从数据源中随机找k个数据点作为质心；
3. 对数据源中的每个数据点，计算到每一个质心的距离，并选择距离最近的质心初次分类；
4. 此时每一个质心下都跟随这一些数据点，通过这些数据点重新选择新的质点；
5. 如果新旧质心之间的距离小于某一个设置的阈值（表示质心位置变化不大，趋于稳定），可以认为聚类已经达到期望，算法终止；
6. 如果新旧质心之间的距离大于阈值，则迭代3-5步骤计算；

### 算法代码

```ts

type Point = Array<number>
const kMeans2 = <T extends Point>(data: T[], k = 1) => {
  // 初始k个质心
  const centroids = data.slice(0, k)
  // 初始化距离数组
  const distances = Array.from({ length: data.length}, () => Array.from({length: k}, () => 0))
  // 类别定义
  const classes = new Map<T, T[]>()
  // 初始化阈值
  const threshold = 0.1
  
  // 迭代控制
  let iterator = true
  while(iterator) {
    iterator = false
    // 初始化类别
    classes.clear()
    for (let idx in centroids) {
      classes.set(centroids[idx], [])
    }

    for (let idx in data) {
      // 计算出当前到每个质心的距离
      for (let c = 0; c < k; c ++) {
        distances[idx][c] = Math.hypot(
          ...Object.keys(data[idx]).map(key => data[idx][key] - centroids[c][key])
        )
      }
      // 计算当前点到最小质心的索引
      const min = distances[idx].indexOf(Math.min(...distances[idx]))
      const points = classes.get(centroids[min])
      points.push(data[idx])
    }
    // 重新计算质心
    let oldCentroids = [...new Set(centroids)]
    for (let c = 0; c < k; c ++) {
      centroids[c] = Array.from({length: data[0].length}, () => 0) as T
      // 计算到当前质心距离最短的点的数量
      const size = data.reduce((acc, point, idx) => {
        const points = classes.get(oldCentroids[c])
        if (points.includes(point)) {
          // 点距离质心距离最短，数量+1
          acc ++
          // 同时计算新质心的位置
          for (let idx in point) {
            centroids[c][idx] += point[idx]
          }
        }
        
        return acc
      }, 0)

      // 新的质点需要取均值
      for (let key in centroids[c]) {
        centroids[c][key] = parseFloat(Number(centroids[c][key] / size).toFixed(2))
      }

    }
    // 比较新旧质心变化是否超过最大阈值
    for (let c = 0; c < k; c ++) {
      const diff = Math.hypot(
        ...Object.keys(oldCentroids[c]).map(key => oldCentroids[c][key] - centroids[c][key])
      )
      if (diff > threshold) {
        iterator = true
        break
      }
    }
  }
  return classes
}
```

下面我们分别用二维平面点以及三维空间点测试一下：
```ts
kMeans([[0, 0], [0, 1], [1, 3], [2, 0]], 2)
// output:
// Map(2) {
//   [ 1, 0 ] => [ [ 0, 0 ], [ 2, 0 ] ],
//   [ 0.5, 2 ] => [ [ 0, 1 ], [ 1, 3 ] ]
// }
kMeans([[0, 0, 0], [5, 0, 100], [6, 0, 0], [8, 0, 0]], 2)
// output:
// Map(2) {
//   [ 4.67, 0, 0 ] => [ [ 0, 0, 0 ], [ 6, 0, 0 ], [ 8, 0, 0 ] ],
//   [ 5, 0, 100 ] => [ [ 5, 0, 100 ] ]
// }
```

## 使用场景
前面我们提到过，`K-Means`不在机器学习场景下不怎么能遇到，那么它的使用场景是什么呢？这里列举一个：

当我们在做D2C（将设计稿生成代码）时，我们期望生成的代码是有布局特性的，比如下面这张图片:
![](./assets/1.jpg)

我们期望是生成的代码是具有弹性布局的，也就是说头像和信息一组、状态和按钮一组，随着客户端机型弹性拉伸，也就是`css` `flex`布局中的`space-between`特性。

然而从设计稿上获取到的数据确是`[img, info, status, button]`这样平铺下来的。因此这种场景下需要使用`K-Means`进行聚类计算，将平铺的数据分为两类`[[img, info], [status, button]]`，然后再适配布局即可实现弹性伸缩。
